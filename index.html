CAMARADES Text Mining

Text-mining script for publication classifications.

Download .zip Download .tar.gz View on GitHub

Background

The screening of current systematic review and meta-analysis relies mainly on human force. While specialists screening provides decent result and double-screening, during which two person screen the same set of publications independently, reduces the error less than 5%, automatic screening is urged to make the publication classification procedures more reproducible and free specialists from repetitive work.

Aim

The goal of this project is to automate the procedure for publication classification. The ultimate goal is to have a live procedure that will pull data from the internet routinely, process them and classify and label them automatically for different aspects. Our first attempt is to have an application to identify Neuroscientific Pain related publications from internet sources, such as PUBMeds, by screening the abstracts of the publications.

Data

Training data are provided by Gillian Currie and Nicki Sherratt in excel format. Dataset from Gillian provides the basic information of the paper and the manual screening decisions. Dataset from Nicki includes information of the article type from ENDNote, which is used in elimite the conference abstract in the data pre-processing stage . Only the result in agreement is used (27467 instances).

Methods

We used R program in implementing the text-mining procedure. Step 1 pre-processes include transform data into tab separated txt file manually, read txt file and merge the two datasets, clean up the data by excluding the conference abstract and those instance with no abstract in. Step 2 pre-processing after turn the file into corpus is a standard cleanup is done to remove numbers, punctuation and extra spaces, as well as stop words. Words are stemmed as well. We test Naive Bayes, kNN and SVM machine learning models in current code. Other possible models are Decision Trees, Outliers, adaboost, JRip, ‘penalizedSVM’ and etc.
